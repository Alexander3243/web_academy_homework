1) Створити можливість реєстрації , авторизації (запис та читання з json файлу).
2) Зберігати дані про користувача у форматі json.
3) Для неавторизованого користувача - калькулятор має чотири функції: + - * /.
4) Для авторизованого ще + синус косинус тангес котангенс.
5) Авторизований користувач може переглянути історію своїх операцій за останній день (або всю історію).
6) Історію всіх операцій пропоную зберігати у форматі JSON. Поля для операції - id(унікальне число), date - дата операції, operation - сама операція
	Пример:
	 [{

	id : 1,
	date: '20-10-1953',
	operation: '2+2=4',
	user_id: <user_id> (взять значення з файлу users.json)
}]

Для авторизації використовувати логін та пароль.
